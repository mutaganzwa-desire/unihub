# UniHub

A production-oriented Flutter + Firebase mobile platform that connects
university students with student-led startups for internships and practical
work experience inside a university innovation ecosystem.

Two roles share one app:

- **Students** build a profile, discover and filter opportunities, get
  tailored recommendations, apply, track application status in real time, and
  message founders.
- **Startups** create a verified profile, post and manage internships, review
  and decide on applicants, chat with candidates, and read analytics.

## Highlights

- Clean Architecture (domain / data / presentation) with a feature-first
  folder layout.
- Riverpod for state management and dependency injection.
- GoRouter with role-based redirect guards.
- Firebase Auth, Cloud Firestore, Storage, Cloud Messaging, Analytics and
  Crashlytics.
- Realtime chat with typing indicators, read receipts and attachments.
- Content-based recommendation engine (explainable, client-side).
- Startup analytics with charts (views, applications, conversion, trends).
- Offline-first (Firestore cache + connectivity banner).
- Secure Firestore & Storage rules with role enforcement.
- Unit, repository and widget tests.

## Getting started

```bash
flutter pub get
# Configure Firebase (see docs/firebase_setup.md)
dart pub global activate flutterfire_cli
flutterfire configure
flutter run
```

The app intentionally refuses to start until `firebase_options.dart` is
generated by `flutterfire configure` — this prevents shipping against a
misconfigured backend.

## Project structure

```
lib/
  core/            # cross-cutting: theme, router, services, widgets, utils
  features/
    onboarding/    auth/       profile/     internships/
    applications/  bookmarks/  chat/        notifications/
    dashboard/     analytics/  admin/
  app.dart         bootstrap.dart   main.dart
firestore.rules    storage.rules    firestore.indexes.json
test/
```

Each feature follows the same internal split:

```
feature/
  domain/         # entities + repository contracts (no Flutter/Firebase)
  data/           # repository implementations + Firestore mappers
  presentation/   # providers (Riverpod) + screens + widgets
```
